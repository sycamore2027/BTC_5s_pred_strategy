Strategy Logic

Entry Orders

Signal

We define the alpha signal as α = pmodel - pmarket. In code, p_model is the model-implied fair value of the UP contract 5 seconds ahead, and p_market is the current UP mid price.

If α > 0.04: Buy UP
If α < -0.04: Buy DOWN
Else: Stay flat

Parameter Choice

We selected the entry threshold by validation search. The chosen value is large enough to filter micro-noise but still frequent enough to produce a meaningful trade sample.

  • Threshold tuned on validation, then frozen for the test set.
  • Higher theta reduced churn and fee drag.
  • Position size stays fixed at $100 notional per trade.

Market Phenomenon

We are trying to capture delayed probability updates, thin liquidity, and short-lived order-book imbalances. The bet is not on the final 15-minute outcome itself; it is on the market repricing toward fair value over the next few seconds.

Monitoring & Failure Detection

How We Know When the Strategy Is Working

On-track Conditions

Going forward, we compare live rolling performance to the validation window used to select the parameters. If rolling Sharpe, rolling expected return per trade, and rolling win rate remain above their validation floors, the strategy is behaving in line with the backtest.

Failure Conditions

We treat the strategy as broken if rolling performance drops below those floors, if returns stay negative for an extended block of trades, or if fee-adjusted PnL keeps decaying. When triggered, the next actions are to reduce size, pause trading, and retrain or recalibrate the signal model.

  • Rolling Sharpe below benchmark floor.
  • Rolling expected return below benchmark floor.
  • Rolling win rate below benchmark floor.
